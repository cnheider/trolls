reset()[source]¶

Reset all the environment_utilities and return an array of observations, or a tuple of observation arrays. If step_async is still doing work, that work will be cancelled and step_wait() should not be called until step_async() is invoked again.

seed(seed=None)¶

Sets the seed for this env’s random number generator(s).

Note

Some environments use multiple pseudorandom number generators. We want to capture all such seeds used in order to ensure that there aren’t accidental correlations between multiple generators.

Returns

Returns the list of seeds used in this env’s random

number generators. The first value in the list should be the “main” seed, or the value which a reproducer should pass to ‘seed’. Often, the main seed equals the provided ‘seed’, but this won’t be true if seed=None, for example.

Return type

list<bigint>

step_async(actions)[source]¶

Tell all the environment_utilities to start taking a step with the given actions. Call step_wait() to get the results of the step. You should not call this if a step_async run is already pending.

step_wait()[source]¶

Wait for the step taken with step_async(). Returns (obs, signals, terminals, infos): - obs: an array of observations, or a tuple of arrays of observations. - signals: an array of rewards - terminals: an array of “episode terminal” booleans - infos: a sequence of info objects
